Welcome to Echo Publish

Echo Publish is a web service for generating and publishing formatted documents through the merging of templates and data. Manage your company’s outgoing mail with ease.

Templates + Data = Documents

It really is that simple.

  1. Create templates in using familiar tools - at the moment this means Google Docs, but more options are planned.
  2. Call the Echo Publish service, providing the data you need to be merged into your templates.
  3. Receive links to the finished documents. They can be linked to, archived, downloaded or whatever you choose.

Test

The test page provides a test harness and allows you to test your templates using test data files or suitable payloads. The page looks like this:

The form in the top part of the page allows for navigation through your template folders. Use the dropdown to choose the folder to navigate to. The lower form offers (in the initial state) three input fields.

  • Choice of templates

    A template is a document or a document structure that typically contains placeholder or tags that define how the supplied test data should be merged to create the finished documents. You can create these in your Google Drive folder.

  • Choice of test data files

    Usually the data that is merged into templates will be supplied dynamically through the merge request that your systems will issue to Echo Protect. For testing purposes, it will be useful to have predefined data sets in xml or json form to merge with your templates. You can create these in your Google Drive folder.

  • Choice of flow

    A flow definition is a set of steps that Echo Publish will carry out when the merge process is called. For example the md.flo definition is used to merge markdown *.md files, and (in the default configuration) will do the following steps: dump the data file supplied as a dictionary for debug purposes; merge the data with the template to provide a merged markdown file; convert the markdown file to html. Other flows will contain other steps, and you can create your own using Echo Publish’s collection of basic steps, which contains merge steps, transformation steps, upload and download options, and more.

Try it out: use the top form to navigate to “Demo Examples”, then try using the Fact Sheet template with the “FS_Devs.xml” test data. This is a docx template, so use the docx.flo flow definition.

Library

The Library page is where you can see Echo Publish resources. It reflects the state of your Google Drive folders, but also shows which of those resources Echo Publish has itself downloaded for efficiency of access, and whether or not those are fresh or not. Each type of asset has a Refresh button displayed: clicking this will cause Echo Publish to fetch the latest version from your Google Drive account. (This will mean that it does NOT have to make a call to Google Drive to fetch the resource when merging, and so will be more efficient.)

The library holds the following types of resource:

  • Templates

    Documents or a document structures that define the finished documents.

  • Flows

    JSON files that define steps in the merge process.

  • Transforms

    XSLT transformation definitions that can be used on XML input data.

  • Test Data

    XML or JSON files to supply data for testing the templates.

  • Branding

    XML or JSON files that supply additional data to be incorporated into the documents. As the name suggests, “branding” files can supply data that can turn generic “white-label” templates into “branded” templates by incorporating company names, contact details, logos, colour schemes and so on.

Archive

The archive page provides access to the 10 most recent merge requests and the most recently produced documents, each list sorted to show most recent first.

  • Merge Requests

    The file name will indicate success or failure. Clicking on the link provided will show the input parameters of the request, and the detailed outcome.

  • Documents

    The documents produced by merge requests. Clicking on the link provided will either show or download the document (according to document type).

Account

The account page shows/will show details of the user’s account.

It also allows for download of the resources into a zip file for backup purposes, and for upload of a previous backup zip file.

Links

Links to access Echo Publish help files and other support materials.
